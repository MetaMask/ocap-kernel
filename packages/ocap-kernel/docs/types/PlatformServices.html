    Type Alias PlatformServices

    A service for things the kernel worker can't do itself. Abstracts platform-specific details of how vat workers are launched, terminated, and connected to the kernel, and for network communications.

          The seed for generating this kernel's secret key.

        • options: RemoteCommsOptions

          Options for remote communications initialization.

          Options for initializing remote communications.

          • OptionalcleanupIntervalMs?: number

            Interval in milliseconds between stale peer cleanup runs (default: 15 minutes). Controls how often the system checks for and removes stale peer data.

          • OptionalmaxConcurrentConnections?: number

            Maximum number of concurrent connections (default: 100). When the limit is reached, new outbound connections are rejected with ResourceLimitError and inbound connections are closed immediately.

          • OptionalmaxMessageSizeBytes?: number

            Maximum message size in bytes (default: 1MB). Messages exceeding this limit are immediately rejected with ResourceLimitError before any connection or queuing attempt.

          • OptionalmaxQueue?: number

            Maximum number of messages to queue per peer while reconnecting. If not provided, uses the default MAX_QUEUE value (200).

          • OptionalmaxRetryAttempts?: number

            Maximum number of reconnection attempts. 0 = infinite (default). If not provided, uses DEFAULT_MAX_RETRY_ATTEMPTS.

          • Optionalrelays?: string[]

            Array of relay peer IDs/multiaddrs to use for remote communications.

          • OptionalstalePeerTimeoutMs?: number

            Time in milliseconds before a disconnected peer is considered stale (default: 1 hour). When a peer has been disconnected for longer than this duration and is not actively reconnecting, its data is cleaned up including: message queues, location hints, connection timestamps, and reconnection state.

        • remoteMessageHandler: RemoteMessageHandler

          A handler function to receive remote messages.

        • OptionalonRemoteGiveUp: OnRemoteGiveUp

          Optional callback to be called when we give up on a remote.

        Returns Promise<void>

        A promise that resolves once network access has been established or rejects if there is some problem doing so.
