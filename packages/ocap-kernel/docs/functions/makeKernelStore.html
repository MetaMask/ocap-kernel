      Create a new KernelStore object wrapped around a raw kernel database. The resulting object provides a variety of operations for accessing various kernel-relevent persistent data structure abstractions on their own terms, without burdening the kernel with the particular details of how they are represented in storage. It is our hope that these operations may be later reimplemented on top of a more sophisticated database layer that can realize them more directly (and thus, one hopes, more efficiently) without requiring the kernel itself to be any the wiser.

        The kernel database this store is based on.
