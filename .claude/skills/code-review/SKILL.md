---
name: code-review
description: How to review code; a pull request, feature branch, local changes etc.
---

When asked to review code, whether a PR on GitHub or some local changes, provide feedback on:

- Code quality and best practices
- Potential bugs or issues
- Performance considerations
- Security concerns
- Test coverage

Use the repository's CLAUDE.md for guidance on style and conventions. If you
encounter terms with definitions particular to this project, consider adding
them to the glossary. Be constructive and helpful in your feedback.
